### 1. - help

~~~
docker command --help
eg:  docker ps --help

Usage:  docker ps [OPTIONS]

List containers

Options:
  -a, --all             Show all containers (default shows just running)
  -f, --filter filter   Filter output based on conditions provided
      --format string   Pretty-print containers using a Go template
  -n, --last int        Show n last created containers (includes all states) (default -1)
  -l, --latest          Show the latest created container (includes all states)
      --no-trunc        Don't truncate output
  -q, --quiet           Only display container IDs
  -s, --size            Display total file sizes

~~~

### 2. systemctl 接管

- docker开关
- systemctl

~~~docker
启动docker
systemctl start docker

重启
systemctl restart  docker

关闭docker
systemctl stop docker
systemctl stop docker.socket

查看docker的运行状态
systemctl status docker
~~~

### 3. docker原生命令

- 容器

~~~docker
- 启动
docker run imagename command
eg: docker run ubuntu:15.10 /bin/echo "Hello world"
- 参数说明
- -i允许你对容器内的标准输入STDIN进行交互
- -t在新容器内制定一个伪终端（terminal）
- -d 后台运行
- -p  指定端口启动 宿主机端口:内部端口
- -P:将容器内部使用的网络端口随机映射到我们使用的主机上。
- --name string  Assign a name to the container
- 
以指定端口启动
docker run --name mavendemo-3 -p 8080:8080 -d mavendemo-3

- 启动已停止的容器
docker start containerName/id 

- 日志
docker logs containerName/id

- 列出容器
正在运行的
docker ps -a
包括未运行的
docker ps

- 进入容器
docker attach
docker exec：推荐大家使用 docker exec 命令，因为此退出容器终端，不会导致容器的停止。
详细说明 docker exec --help

- 停止
docker stop containerName/id

- 重启
docker restart <容器 ID>


- 导出容器
docker export d85208b229a9 > /tmp/hello.tar
- 导入容器快照
cat /tmp/hello.tar | docker import - test/hello:v1
也可以指定url来导入
docker import http://example.com/exampleimage.tgz example/imagerepo

- 删除
docker rm -f 1e560fca3906

- 运行web应用
- 载入镜像
docker pull training/webapp
docker run -d -P training/webapp python app.py
docker run -d -p 80:5000 training/webapp python app.py

- 端口
查看端口映射
docker port bf08b7f2cd89

- 容器内进程
docker top containerName/id

~~~

- 镜像

~~~
- 列出已有的镜像
docker images 


~~~

- 登录和退出

~~~
docker login 
docker logout

- 推送
docker tag imageName username/imageName
~~~
